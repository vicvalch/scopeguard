# Purpose

Meetings is PMFreak's operational cadence module for planning, running, and closing high-value sessions that drive decisions, accountability, and delivery movement.

# Core Entities

- Meeting
- Agenda Item
- Decision Log
- Action Item
- Follow-up Signal

# Primary Workflows

- Prepare focused agendas linked to current project and risk context.
- Capture decisions, owners, and due dates in real time.
- Track follow-through and close loops between meetings.

# AI Behaviors

- Generate agenda priorities from live operational signals.
- Summarize discussions into decisions and concrete actions.
- Detect recurring unresolved items and recommend escalation.

# Operational Signals

- Action-item completion lag.
- Decision churn without resolution.
- Repeated meeting topics indicating structural blockers.

# UX Notes

The UX should minimize ceremony and maximize operational throughput. Users should quickly move from preparation to execution to accountable follow-up.

# Visual Direction

Chronological meeting streams, actionable checklists, and decision snapshots. Interfaces should feel disciplined, lightweight, and outcome-centered.

# Future Runtime Behaviors

Runtime intelligence should auto-compile briefing packets, detect execution risk from meeting outcomes, and trigger cross-module updates after decisions are confirmed.
