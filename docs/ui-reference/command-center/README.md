# Purpose

The Command Center is PMFreak's mission control surface. It gives PMO leaders and delivery owners a prioritized, cross-portfolio view of what requires action now.

# Core Entities

- Alert
- Priority Queue Item
- Program Health Snapshot
- Escalation
- Action Recommendation

# Primary Workflows

- Review daily priority queue generated from portfolio risk and delivery movement.
- Triage escalations by urgency, impact, and dependency blast radius.
- Launch guided actions that route users into the correct project, risk, or stakeholder context.

# AI Behaviors

- Continuously rank signals by operational impact.
- Summarize why each item is critical in plain operational language.
- Recommend the next best action with confidence and expected outcome.

# Operational Signals

- Milestone slippage across active programs.
- New blockers, unresolved dependencies, and trend reversals.
- Escalation velocity and unresolved critical-age items.

# UX Notes

The interaction hierarchy should feel decisive: urgent first, context second, deep analysis on demand. The emotional tone should reduce chaos and increase executive confidence.

# Visual Direction

High-contrast status surfaces, clear severity coding, and compact cards with expandable detail. Motion should reinforce priority transitions and state changes.

# Future Runtime Behaviors

The runtime should orchestrate autonomous monitoring agents that raise alerts, correlate risks across modules, and trigger workflow automations when thresholds are crossed.
