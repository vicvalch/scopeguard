# PMFreak UI Reference

PMFreak uses Bubble-generated UX exploration as a canonical visual and product reference for experience design decisions. Bubble is used to validate interaction patterns and operational flow concepts quickly, but it is **not** the production runtime.

The real implementation target is the PMFreak application stack:

- Next.js
- Tailwind CSS
- shadcn/ui
- Framer Motion
- PMFreak runtime architecture

This `docs/ui-reference/` system exists to:

- preserve UX consistency across modules and delivery cycles
- align Claude and Codex implementation behavior around one source of product truth
- avoid design drift during iteration and feature expansion
- define PMFreak's canonical operational intelligence experience

Each module folder captures intent, workflows, AI behavior, and operational signals so engineering and product decisions remain implementation-oriented while preserving the platform's operational intelligence positioning.
